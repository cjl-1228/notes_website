第一章

  • 雲端運算的特色
    • 自主服務

    • 快速彈性
    • 按量計價
  • 雲端服務的類型與代表服務供應商
    • 基礎設施即服務(IaaS)
      • 提供計算、儲存裝置、通訊等資源的虛擬化或虛擬機
      • 供應商：Amazon EC2、Google Compute Engine、Rackspace Cloud、Flexiscale、GoGrid、Joyent Cloud、RightScale
    • 平台即服務(PaaS)
      • 提供開發的平台，包含作業系統、程式語言的執行環境、資料庫、網頁伺服器等
      • 供應商：Amazon Elastic Beanstalk、Google App Engine、Microsoft Windows Azure、Heroku、Forec.com、EngineYard、Mendix
    • 軟體即服務(SaaS)
      • 應用程式在雲端上安裝和執行、而雲端的使用者透過終端使用軟體，而無需管理相關的基礎設施和平台
      • 供應商：Google Apps、Microsoft Office 365、Saleforce.com、Quickbooks Online
  • 雲端部署的模式與其特性/優缺點
    • 私有雲(Private Cloud)
      • 也稱為內部雲(Internal Cloud)，為了一個組織自己使用所設計的雲端
      • 優點：效率、可靠度、安全有高度的控制能力
      • 缺點：需要自己購買、建置、管理，沒有初期無需建置成本的好處
    • 公有雲(Public Cloud)
      • 提供計算、儲存和應用程式資源給一般大眾使用，通常是由一個大型組織所提供，用戶透過Internet來存取服務，並採用隨用隨附的收費方式
      • 優點：初期不用大量投資成本在基礎設施
      • 缺點：缺乏較好的控制資料、網路和安全的機制，所以在企業的運用中大大降低效能
    • 混合雲(Hybrid Cloud)
      • 結合公有雲和私有雲，保有各自的特點又採用所有的優點。一部分的服務在公有雲中執行，一部分在私有雲中執行
      • 優點：比公有雲和私有雲彈性，比公有雲具有更緊密的控制和安全性，可以有較佳的隨取即用的好處
      • 缺點：在設計混合雲時需要仔細考慮公有雲和私有雲結合的分割點
    • 社群雲(Community Cloud)
      • 一些組織有共同特定的考量(例如任務、安全需求、政策法規)共享基礎設施，可能由自己、第三方、或是外部供應商來管理
      • 優點：共享資源與安全性可同時兼顧、費用比私有雲少
      • 缺點：費用比公有雲多
  • 雲端運算重要的技術
    • 硬體
      • 虛擬化技術
        • 在資料中心建置和維運操作是一個好的解決技術
        • 將處理器、記憶體和I/O設備等資源，增進資源的分享和使用率
        • 可以在一個實體機器上執行許多的作業系統和軟體
    • 分散式計算
      • 網格計算
        • 利用大量異構計算機的未用資源(CPU和磁碟儲存)，作為分散式網路中的一個虛擬計算叢集，為解決大規模的計算問題提供一個模型
        • 鬆散耦合、異質性和分散的特性
      • 公用計算
        • 整合計算資源，例如CPU、儲存裝置和服務，而且可以計量的服務。公用計算經由大量的電腦能夠支援大量或是突然巨量計算的需求
    • Internet技術
      • Web服務(Web Service, WS)
        • 可以在不同的訊息的產品平台上將應用軟體整合起來，使得在一個應用程式資訊可以在另外一個應用程式上使用，也可以使得內部的應用程式可以透過Internet使用
        • WS是一個軟體系統，用於支援網路間不同機器的互動操作，通常是許多API所組的
      • 服務導向架構(Service-oriented Architecture, SOA)
        • 是一種軟體開發方法，它使用稱為服務的軟體元件來建立業務應用程式。每個服務都提供一種業務功能，服務之間還可以跨平台、跨語言進行通訊。開發人員使用 SOA 在不同系統中重複利用服務，或組合幾個獨立的服務來執行複雜的任務
        • 例如，某個組織的多個業務流程需要使用者身分驗證功能。該組織無需為所有業務流程重寫身分驗證程式碼，而只需建立一個身分驗證服務，再將該服務用於所有應用程式。同樣的，醫療機構中的幾乎所有系統 (例如患者管理系統和電子健康記錄 (EHR) 系統) 都需要對患者進行登記。這些系統可以呼叫單個通用服務來執行患者登記任務。
      • Web 2.0
        • 使用者可以透過虛擬社群中與使用者產生內容的互動
    • 系統管理
      • 自主計算(Autonomic Computing)
        • 減少人為的操作來改進系統的效率。自主計算使用高階策略來自我決策，將持續檢查以達最佳化的狀態，自動自我調整去合適現況。
  • 何謂服務等級協定(SLA)與其好處
    • 雲端運算的服務就像是公共資源的使用一樣，不同的服務品質(QoS)必須保證能符合使用者的需求
    • SLA是服務供應商和使用者間確保服務品質的正式契約
    • 優點：
      • 加強客戶的滿意程度
      • 改善服務品質
      • 改善雙方關係
  • 雲端運算的挑戰，舉2個說明之類
    • 安全、隱私和法規
      • 公有雲屬於暴露的系統會帶來更多的攻擊機會，如何讓雲端系統環境的安全性像本機端的系統安全
      • 雲端運算使用了許多第三方(third-party)的服務或設施去處理相關資料或運算，會對使用者的資料或相關技術造成隱私問題
      • 雲端的提供者可以選擇全球任一地方來儲放資料，但實際資料中心會受到當地法規限制
    • 資料套牢和標準
      • 雲端的使用者會擔心他們的資料會被某些雲端提供者套牢
      • 若服務不符合使用者需求，想將資料或運算移出雲端，但雲端的設施或平台並沒有提供標準的方法去儲存。不具通透性，資料也沒辦法任意轉移
      • 需制定雲端的標準，希望廠商可以無間隙的一起合作
    • 可用性、容錯和回復
      • 使用者希望他們的內容放在雲端環境中，可以得到某些服務品質的保證
    • 資源管理和能源效益
      • 如何有效管理虛擬資源池，虛擬機如何在一個實體資源中考慮一些因素找到好的對應方式，讓使用率最大化
      • 資料中心消耗大量的能源，包含電力、冷卻系統、二氧化碳等

第二章

  • 雲端運算特殊成本，舉2個說明
    • 轉移或更改系統的花費
      • 將現有系統遷移到雲端可能需要重大的資本支出，包括硬體和軟體的調整、設定這也包括可能的人力資源培訓和遷移成本。
    • 資料安全的問題
      • 資料安全是組織在雲端運算中的關鍵考慮因素。組織可能需要投資於加強雲端環境的安全性，這包括加密、身份驗證、訪問控制等安全措施。這些安全措施可能需要額外的成本。
    • 當前雲端平台的使用限制
      • 不同的雲端供應商和服務可能有使用限制，如最大存儲空間、計算資源限制等。組織可能需要根據其需求支付額外費用以擴展這些限制。
    • 雲端服務轉移至另一個雲端服務或是私有設備的成本
      • 如果組織決定將雲端服務轉移到另一個供應商或是回歸到私有設備，這可能需要額外的成本。這包括設置新的環境、資料遷移、執行成本和合規性考慮。
    • 傳統系統的授權機制與雲端環境不同所增加的授權成本
      • 傳統系統的授權模型和雲端環境可能不同。組織可能需要重新考慮其軟體授權和許可成本，以確保符合雲端環境的需求。
  • 雲端運算對中、小型組織的優點，舉2個說明
    • 靈活性
      • 中小企業對於市場敏感且需快速切入市場，雲端允許企業快速佈建且建立服務
    • 成本
      • 雲端的月多少付多少特性能夠在初期幫助中小企業無需投入大量IT成本，可將這些資源放入更有用的開銷上，例如研發產品與廣告等
    • 服務使用性
      • 雲端可確保服務是穩定正常運行的，可節省管理成本
      • 中小型企業難以建立完善的備份機制，可透過雲端來備份
      • 若組織發展成功，雲端的自動擴轉(Auto Scaling)可以自行定義如何擴展以滿足成長需求
    • 人事成本
      • 雲端需要高度且專業的程式開發人員，會比傳統的服務花費更多的成本，但相對的，在雲端環境的維運成本則大幅減少，且能源消耗也由供應商分擔，企業負擔減少
  • 雲端運算對大型組織的優點，舉2個說明
    • 靈活性
      • 大型組織在建立或是發展一個新服務的時候需要注意軟體整合及資料傳輸的狀況，否則會導致成本不符預期，如果使用雲端的話失敗也不會有太多的固定成本需要負擔
    • 成本
      • 敏感資料、內部資料、個資可放在計有的IT設備
      • 需要用到大量運算資源，但不屬於上述關鍵資料則可改成雲端方式節省成本
    • 軟體
    • 延展性
    • 員工量或是技能
    • 備援
